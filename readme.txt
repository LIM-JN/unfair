윷놀이 게임을 웹으로 구현한 프로젝트입니다.

단순한 UI 구현을 넘어, 실제 게임 규칙을 상태 기반으로 설계하여
말의 이동, 분기 경로(지름길), 턴 관리, 추가 턴(윷/모), 잡기, 업기 기능까지 구현했습니다.

특히 여러 말이 하나의 단위로 동작하는 ‘업기’ 기능을 위해
leader / follower 구조를 설계하여 상태를 관리한 것이 핵심입니다.

또한 DOM 위치를 직접 계산하여 말의 이동을 구현하고,
윷 던지기 애니메이션과 턴 표시(UI)를 추가하여 사용자 경험을 개선했습니다.

🤖 AI 플레이어 구현

상대 플레이어(B)는 AI로 동작하도록 구현했습니다.

AI는 단순 랜덤이 아닌 점수 기반 의사결정 방식으로 동작하며,
다음과 같은 기준을 종합적으로 평가하여 최적의 말을 선택합니다.

상대 말 잡기
골(도착) 가능 여부
지름길 진입 여부
안전 구간 여부
위험 구간 진입 / 탈출 여부
아군 말 업기 및 업은 말 우선 이동

또한 이동 시뮬레이션과 거리 계산을 통해
현재 상태와 이동 후 상태를 비교하여 판단하도록 설계했습니다.

⚙️ 주요 구현 포인트

상태 기반 게임 로직 (gameState 관리)
분기 경로를 포함한 윷놀이 보드 구조 설계
leader / follower 구조를 활용한 업기 시스템
이동 시뮬레이션을 통한 AI 판단 로직
BFS 기반 거리 계산을 활용한 위험 구간 판단
사용자 입력(A)과 AI(B) 턴 분리 처리

🎯 특징

단순 구현이 아닌 “게임 규칙의 구조화”에 집중
실제 플레이 가능한 수준의 AI 로직 포함
UI, 로직, 상태 관리가 결합된 프로젝트
